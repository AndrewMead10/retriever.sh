| Segment | Description | Why They Buy |
|---------|-------------|--------------|

## Competitive Differentiation

| Competitor | Their Play | Retriever's Advantage |
|------------|-----------|----------------------|
| **Pinecone** | Enterprise-grade vector DB | Overkill for most. Complex pricing. Retriever is simpler & cheaper. |
| **Weaviate Cloud** | Full-featured, self-hostable | Same as above. Great if search is core to your product - but for most it isn't. |
| **Algolia** | Traditional search, great UI | Expensive at scale. Not optimized for semantic/RAG. |
| **Roll your own** | pgvector + OpenAI embeddings | You're now a search engineer. Retriever handles this for you. |

**Our lane**: Developers who need "good enough" search as a feature, not a product. If search is the main part of your product, you probably need something more robust. But if it's just a feature you'd like to have - a nice-to-have along the way - this is the product for you.

## Integration Experience

The product is designed to be as simple as possible:

1. Sign up at retriever.sh
2. Create a project, get your API key
3. Add 2-3 functions to your code (insert, query, optionally delete)
4. Use `curl`, `requests`, `fetch` - whatever you already have
5. Done. Move on to other features.

No new libraries. No SDKs. No configuration files. Just HTTP calls.
